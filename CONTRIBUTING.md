# Contributing

Thank you for helping improve this curated resource list. Contributions from
the community keep these lists useful, accurate, and up to date.

## What belongs here

* High-quality learning resources with canonical links
* Official documentation and well-maintained open-source projects
* Practical tools, courses, books, and communities
* Resources useful to beginners and experienced practitioners
* Original, concise descriptions written for this list

## What does not belong here

* Spam, affiliate-only pages, or promotional content
* Copied descriptions from other awesome lists
* Broken, dead, or unverifiable links
* Piracy, malware, exploit, or harmful resources
* Duplicate entries without added value
* Separate category Markdown files (all content stays in `README.md`)

## Quality bar

Every resource should score **4 or 5** on usefulness:

| Score | Meaning | Include? |
| ----: | ------- | -------- |
| 5 | Exceptional and widely useful | Yes |
| 4 | Strong and practical | Yes |
| 3 | Useful but limited | Maybe |
| 2 | Narrow, outdated, or shallow | Usually no |
| 1 | Low quality or spammy | No |

## How to submit a resource

Add a row to the appropriate table in `README.md`:

```markdown
| [Resource Name](https://canonical-url) | Type | Level | Cost | One original sentence explaining why it matters. | Tags |
```

Use these levels: **Beginner**, **Intermediate**, **Advanced**  
Use these costs: **Free**, **Paid**, **Freemium**, **Open Source**

## Pull request process

1. Fork the repository
2. Create a branch for your change
3. Edit `README.md` only (unless fixing templates or docs)
4. Use canonical URLs and write original descriptions
5. Open a pull request using the PR template
6. Respond to review feedback

## Review checklist

Before submitting, confirm:

- [ ] Link works and points to the original source
- [ ] Description is original and useful
- [ ] Category and level are correct
- [ ] No duplicate entry exists
- [ ] Resource is relevant and high quality

## Good example

```markdown
| [MDN Web Docs](https://developer.mozilla.org/) | Documentation | Beginner | Free | Canonical reference for HTML, CSS, and JavaScript on the web platform. | 🟢 🆓 🌐 ⭐ |
```

## Bad example

```markdown
| [Cool site](https://example.com) | Website | Beginner | Free | Best site ever!!! | 🔥 |
```

## Attribution

Link to original sources. Do not copy content from other lists. External
resources belong to their respective owners.

## Becoming a maintainer

Consistent, high-quality contributions over time may lead to maintainer access.
Maintainers help review PRs, keep links healthy, and protect list quality.
